## 📐 2. กฎทองสําหรับ Copywriters & Designers (i18n Best Practices)

เมื่อออกแบบหรือเขียนคำอธิบายเนื้อหาบน Figma กรุณาพิจารณา 3 กฎข้อบังคับด้าน i18n ดังนี้:

### ⚠️ กฎข้อที่ 2.1: เฝ้าระวัง "ความยาวข้อความแตกต่าง" (Length Drift Defense)
> [!IMPORTANT]
> โดยธรรมชาติ **ภาษาไทยมักมีความยาวมากกว่าภาษาอังกฤษประมาณ 1.3 - 1.8 เท่า** ในความหมายเดียวกัน

*   **สิ่งที่ดีไซเนอร์ต้องทำ:** กรุณาเผื่อพื้นที่สำหรับการขยายตัวของกล่องข้อความภาษาไทยใน Figma เสมอ (เช่น บน Figma ใส่ภาษาอังกฤษตัวสั้นกระชับยาว 80px แต่พอเป็นภาษาไทยอาจกว้างถึง 150px) Layout ต้องสามารถยืดขยายแนวตั้งหรือขึ้นบรรทัดใหม่ได้สวยงามโดยไม่ทับซ้อนส่วนอื่น
*   **ตัวอย่าง:**
    *   EN: *"Win Now"* (สั้นและกระชับ)
    *   TH: *"ร่วมสนุกเพื่อลุ้นรับรางวัลตอนนี้"* (ยาวกว่าเดิมมาก)

### 🔤 กฎข้อที่ 2.2: กำหนดคีย์พารามิเตอร์ไดนามิก (Dynamic Parameter Interpolation)
บางครั้งข้อความจำเป็นต้องมีตัวแปรเปลี่ยนตามเวลาหรือปี เช่นการดึงปีจากฐานข้อมูลมาแสดงผลในโค้ด:
*   **รูปแบบใน JSON:** `"yearToPresent": "{year} – ปัจจุบัน"`
*   **ดีไซเนอร์ควรระบุ:** ใน Figma ให้ทำเครื่องหมายสำหรับค่าตัวแปร เช่น `[Year] - Present` เพื่อให้ทีมพัฒนาเขียนโค้ดแมปตัวแปรไดนามิกได้ถูกต้องโดยไม่ต้อง Hardcode ปีลงระบบ

### 📂 กฎข้อที่ 2.3: ส่งมอบคู่ภาษาแบบเคียงคู่ (Side-by-Side Copy Delivery)
*   เมื่อคุณจัดทำ Figma Draft สำหรับ Section ใหม่ กรุณาแนบข้อความทั้ง **ภาษาไทย** และ **ภาษาอังกฤษ** ควบคู่กันเสมอ 
*   หลีกเลี่ยงการออกแบบเฉพาะภาษาอังกฤษแล้วเขียนโน้ตว่า *"ภาษาไทยแปลทีหลัง"* เพราะจะทำให้เราวาง CSS Grid และ Line Height สอดรับฟอนต์ Display เสียสมดุลได้ง่าย
